# Gemini Model Function Calling Stress Test

이 프로젝트는 다수의 도구(Tools)가 주어졌을 때 여러 Gemini 모델들이 멀티턴(Multi-turn) 대화 상황에서 질문의 의도에 맞는 정확한 도구를 연속적으로 선택하여 호출(Function Calling)할 수 있는지 평가하는 스트레스 테스트 스크립트입니다. 

특히 비슷한 파라미터 구조를 가진 **함정 도구(Poison Pills)**들을 배치하여 모델의 맥락 이해도 및 멀티턴 도구 선택 정확도를 확인합니다.

## 🚀 주요 기능

- **다량의 더미 도구 생성**: 주어진 갯수만큼 무작위의 영어 도구 이름과 한글 설명을 가진 더미 도구를 동적으로 생성합니다.
- **함정 도구(Poison Pills)**: 정답 결제/예약 도구와 유사하지만 미세하게 다른 기능을 하거나 파라미터 구조를 30% 확률로 동일하게 복제한 함정 도구를 추가하여 모델의 분별력을 깐깐하게 테스트합니다.
- **다중 모델 비교**: 스크립트 실행 한 번에 다음 3가지 모델을 호출하여 결과를 비교합니다.
  - `gemini-3.1-pro-preview`
  - `gemini-3-flash-preview`
  - `gemini-3.1-flash-lite-preview`
- **멀티턴 테스트**: 단순한 1회성 호출을 넘어, 재고 확인(Turn 1, 2) 후 결제(Turn 3)로 이어지는 자연스러운 문맥을 모델이 유지하는지 테스트합니다.

## 📦 설치 및 실행 방법

1. **의존성 설치**: 패키지 관리자인 `uv`를 통해 필요한 패키지를 설치합니다.
   ```bash
   uv add google-generativeai python-dotenv
   ```
2. **환경 변수 설정**: 프로젝트 루트 경로에 `.env` 파일을 생성하고 발급받은 Gemini API 키를 입력합니다.
   ```env
   GEMINI_API_KEY="your_api_key_here"
   ```
3. **스크립트 실행**:
   ```bash
   uv run main.py
   ```

## 📊 재현(Reproducibility) 및 테스트 환경

본 테스트는 외부 제어 요인을 최소화하여 누구나 동일하게 스크립트만으로 재현할 수 있도록 설계되었습니다.

- **실행 환경 및 파라미터**:
  - **테스트 모델**: `gemini-3.1-pro-preview`, `gemini-3-flash-preview`, `gemini-3.1-flash-lite-preview` (Model Config는 기본값 사용)
  - **도구 설정**: `TOTAL_TOOL_COUNT = {입력}`개의 도구가 생성되며, 정답 도구는 도구 리스트의 `middle`(중간)에 배치됩니다.
  - **반복 실행**: 우연에 의한 결과(환각 등)를 방지하기 위해 전체 테스트 스위트를 3회(`range(3)`) 반복 실행합니다.
- **데이터셋 위치**: **외부 데이터셋(JSON, CSV 등)은 필요하지 않습니다.** 모든 더미 도구와 멀티턴 프롬프트(Turn 1~3)는 `main.py` 코드 내부에서 동적으로 생성되고 정의되어 있으며 별도의 파일 다운로드 없이 즉시 실행 가능합니다.

## 🧪 기본 테스트 시나리오 (멀티턴)

- **Turn 1**: 
  - 사용자: *"검은색 무지 티셔츠 L 사이즈 재고 있어?"*
  - 정답 도구: `check_fashion_stock`
- **Turn 2**: 
  - 사용자: *"좋아, 그럼 그거랑 같이 입을 흰색 M 사이즈도 재고 확인해줘."*
  - 정답 도구: `check_fashion_stock`
- **Turn 3**: 
  - 사용자: *"둘 다 장바구니에 넣고 아까 말한 검은색만 10% 쿠폰 써서 바로 결제해줘!"*
  - 정답 도구: `execute_direct_purchase`

스트레스 테스트를 실행하면 각 모델이 연속된 대화 맥락을 유지하면서 올바른 파라미터(`stock`, `color`, `coupon_code` 등)를 추출하여 도구를 호출하는지 콘솔에 결과를 자세히 출력합니다.
